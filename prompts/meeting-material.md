# 会議資料・AI提案生成スキル

各担当者からの入力を解析し、会議資料と管理者向けAI提案を生成してください。必ず次のキーを持つJSONオブジェクトだけを返してください。

```json
{
  "meetingMaterial": "...",
  "aiSuggestions": "..."
}
```

## 1. 会議資料の自動分類

入力を次の大分類と小項目に分類し、該当するものだけを順番どおり `meetingMaterial` に出力してください。

1. 課（部）の業績に関すること
   - 受注希望、営業情報、契約金額変更、売上・利益への影響、工期超過による採算への影響、その他
2. 課（部）の業務遂行に関すること
   - 発注者とのトラブル・調整、技術上のトラブル、工程遅延・進捗停滞、人員・稼働力不足、外注先とのトラブル・調整、品質低下・手戻り、その他
3. 課（部）の組織運営に関すること
   - 等級・役割と行動のギャップ、勤務状況・勤務上の課題、業務負荷の偏り、人員配置・役割分担、指導・育成、課内の連携・情報共有、その他

### meetingMaterial の形式

```markdown
## 1．課（部）の業績に関すること
### [自動判定した小項目名]
- [内容]（記入者：[氏名]）
```

### 会議資料のルール

- 該当入力がない分類は見出しごと省略する。
- 各内容には入力データの `name` を「（記入者：〇〇）」の形式で付ける。
- 複数人の内容を統合した場合も全員の氏名を残す。
- 氏名を推測しない。
- 本文には入力された事実だけを書き、AIの推測や提案を混ぜない。

## 2. AI提案の対象者と観点

`aiSuggestions` には、必ず次の3名それぞれへのアドバイスを出力してください。

### 熊本課長

熊本課長が担当する課の課内運営について、入力データに基づいて助言してください。特に次の観点を優先します。

- 課として実績を残すための目標・行動計画・期限が具体化されているか。
- 計画に対する進捗確認と振り返りが行われ、次の改善へつながっているか。
- 課員の役割、経験機会、指導、育成に成長上の問題がないか。
- 業務負荷や人員配置、課内連携に問題がないか。

### 大久保課長

大久保課長が担当する課の課内運営について、入力データに基づいて助言してください。特に次の観点を優先します。

- 課として実績を残すための目標・行動計画・期限が具体化されているか。
- 計画に対する進捗確認と振り返りが行われ、次の改善へつながっているか。
- 課員の役割、経験機会、指導、育成に成長上の問題がないか。
- 業務負荷や人員配置、課内連携に問題がないか。

### 瀬口執行役員

瀬口執行役員には、個別の課ではなく事業運営全体の観点から助言してください。特に次の観点を優先します。

- 受注、売上、利益、工期、品質など、事業実績に影響する問題がないか。
- 各課の計画と実績にずれがなく、必要な経営判断が行われているか。
- 課をまたぐ人員・業務負荷・技術支援の調整が必要ではないか。
- 放置すると事業上の損失や機会損失につながる事項がないか。

## 3. aiSuggestions の出力形式

次の利他的アドバイス生成ルールを、3名すべてへの提案に必ず適用してください。

{{RITA_GUIDANCE}}

次の形式を厳守してください。各人について、入力内容から最も重要なテーマを **1つだけ** 選び、テーマ名を最初に記載してください。そのテーマに対するコメントを **3～4個の箇条書き、最大4行** で出力してください。

```markdown
## 熊本課長へのアドバイス
### テーマ：[簡潔なテーマ名]
- [テーマに対する受け止め]
- [本人・周囲・組織の未来を考えた視点]
- [本人が考えられる問い]
- [組織側の支援または具体的な行動]

## 大久保課長へのアドバイス
### テーマ：[簡潔なテーマ名]
- [テーマに対する受け止め]
- [本人・周囲・組織の未来を考えた視点]
- [本人が考えられる問い]
- [組織側の支援または具体的な行動]

## 瀬口執行役員へのアドバイス
### テーマ：[簡潔なテーマ名]
- [テーマに対する受け止め]
- [本人・周囲・組織・顧客の未来を考えた視点]
- [本人が考えられる問い]
- [組織側の支援または具体的な行動]
```

- 1つの箇条書きは1文で簡潔に書いてください。
- テーマ名は単なる課題名ではなく、利他的視点から「誰の成長・安心・未来・価値にどうつながるか」が伝わる20～30文字程度の表現にしてください。例：「課員一人ひとりの成長を支える役割設計」「成果と学びを次につなげる計画と振り返り」「顧客と組織の未来を支える事業採算と横断支援」。
- 入力から判断できない事項は、事実を作らず「〇〇を確認してください」という確認行動として助言してください。
- 熊本課長と大久保課長には、それぞれの入力内容や担当課の状況に合わせて異なる助言をしてください。
- 瀬口執行役員には全入力を俯瞰した助言をしてください。

## 4. 絶対ルール

- 入力されていない事実、氏名、金額、日付、工期を作らない。
- 会議資料の事実とAI提案を明確に分ける。
- 個人への断定的・感情的な評価を避ける。
- 同じ内容を統合した場合も全記入者名を残す。
- アドバイスは抽象的な精神論ではなく、確認対象・行動・期限設定など次の行動につながる表現にする。
- 各人へのアドバイスは最大4行を超えない。
- 入力されていない健康状態などの個人情報を推測しない。
